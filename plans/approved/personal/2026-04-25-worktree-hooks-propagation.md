## Context

Last session produced two hook-bypass incidents that share one root cause: `scripts/install-hooks.sh` writes hooks to `$(git rev-parse --git-dir)/hooks` — which on a worktree resolves to `.git/worktrees/<name>/hooks/`, not the shared `.git/hooks/`. Existing worktrees inherit nothing from the main checkout, so any hook installed after a worktree was created is invisible inside it.

Two live failure modes already observed:

1. **Senna 4f5d715e (Rule 19 bypass)** — `pre-commit-plan-lifecycle-guard.sh` (PR #43) was installed on main only; her worktree didn't have it, so the commit-phase defence-in-depth guard never ran when she pre-staged plan moves.
2. **Talon PR #45 (universal identity bypass)** — the orphan-identity hook (PR #41) wasn't present on his worktree, so commits were authored as `orianna@strawberry.local`.

Universal worktree isolation (PR #37) generalised the problem: every breakdown/test-plan subagent now spawns into a fresh worktree, none of which inherit hooks. Every new hook we ship needs to magically land in ~10 stale worktree dirs and every future ephemeral one — not viable as-is.

**Fix:** set `core.hooksPath = scripts/hooks-dispatchers/` repo-wide so all worktrees share one hooks directory. This is the documented git mechanism for exactly this case, and `install-hooks.sh` already honors `core.hooksPath` if set (lines 40-43). Belt-and-suspenders: backfill stale worktrees by re-running `install-hooks.sh` is no longer necessary once `core.hooksPath` is set, because the dispatchers move outside `.git/`. Add a one-line check in `safe-checkout.sh` and a `scripts/worktree-add.sh` wrapper to ensure new worktrees pick up the config (they will automatically — `core.hooksPath` is a repo-wide config, not per-worktree — but the wrapper centralises the call site for future hook needs).

## Decision

**Architecture:** single mechanism — `core.hooksPath` pointing to a tracked, in-repo dispatcher directory. Rejected alternatives:

- **Symlink iteration over `.git/worktrees/*/hooks/`** — works today but breaks on every new worktree; requires either a post-checkout hook (which itself has the same install problem) or constant manual re-runs. Defence-in-depth-only would be acceptable but redundant once `core.hooksPath` is set, so we don't ship it.
- **Both** — pure cost, no marginal safety. The `core.hooksPath` mechanism is a single git config value tracked in `.git/config`; if it's missing the hooks don't run anywhere (loud failure mode), which is preferable to silent partial coverage.

**Dispatcher location:** `scripts/hooks-dispatchers/` (new, tracked). Holds the three dispatcher files (`pre-commit`, `pre-push`, `commit-msg`) generated by `install-hooks.sh`. Tracking them is fine — they're generated artifacts but stable and reviewable; the bashlex pip install side-effect remains in `install-hooks.sh`.

**Migration:** one-time `git config core.hooksPath scripts/hooks-dispatchers` set by re-running `install-hooks.sh`, plus a fresh dispatcher write into the new location. Stale `.git/worktrees/*/hooks/` directories become inert (git ignores them once `core.hooksPath` is set) — no cleanup required, but T5 documents the pruning.

## Test plan

Tests protect three invariants:

- **INV-1: hook coverage in worktrees.** A pre-commit guard installed on main MUST fire when invoked from any worktree of the same clone. T1 covers this end-to-end by exercising `pre-commit-plan-lifecycle-guard.sh` from inside a freshly-`git worktree add`'d directory; the test fails on current main and passes once T2 lands.

- **INV-2: idempotent install.** Running `install-hooks.sh` twice MUST NOT corrupt the dispatchers or leave `core.hooksPath` unset. Add an assertion at the end of T1's script (or as a sibling test) that runs `install-hooks.sh` twice and verifies `git config --get core.hooksPath` is unchanged and the three dispatcher files have identical content across runs.

- **INV-3: fail-loud on unset config.** `scripts/worktree-add.sh` MUST refuse to create a worktree when `core.hooksPath` is unset. Cover with a one-shot bash test in `scripts/hooks/tests/test-worktree-add-wrapper.sh` (new) that unsets `core.hooksPath` in a temp clone, invokes the wrapper, and asserts non-zero exit + an error message mentioning `install-hooks.sh`.

Test-plan-only invariant (no test required, asserted in review): T2 must NOT delete or move existing `.git/hooks/` content. The dispatchers there are stale-but-harmless once `core.hooksPath` is set; deleting them risks breaking any external tool that pokes at `.git/hooks/` directly.
