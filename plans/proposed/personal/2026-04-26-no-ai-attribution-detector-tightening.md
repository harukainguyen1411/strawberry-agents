## Context

The no-AI-attribution defense (Rule 21) currently matches **bare strings** for model
names — `Claude`, `Anthropic`, `Sonnet`, `Opus`, `Haiku` — anchored only by punctuation
boundaries. In a codebase whose literal vocabulary contains "Sonnet" and "Opus" (agent
model tiers, routing tables, memory shards, plan frontmatter), this fires false positives
on legitimate technical mentions. The escape hatch — `Human-Verified: yes` — was meant
for rare human pair-programming attribution but has been repeatedly pasted into commits
to push past those false positives. Discipline erosion. Duong's directive: remove the
override, fix the detector to match attribution **context**, not bare strings.

Decision (Duong, 2026-04-26): leave history alone, purge the trailer mechanism going
forward, no tripwire. Orphan `Human-Verified:` trailers in past commits become inert.

Two artifacts change behaviorally — `scripts/hooks/commit-msg-no-ai-coauthor.sh` and
`scripts/ci/pr-lint-no-ai-attribution.sh`. Two artifacts change documentationally —
`.claude/agents/_shared/no-ai-attribution.md` (which then propagates via
`scripts/sync-shared-rules.sh`) and `CLAUDE.md` Rule 21.

## Detection rules (post-change)

**Always block (any context):**
- `Co-Authored-By:` trailer with any value (universal).
- AI email domains: `@anthropic.com`, `@claude.com`, `@noreply.anthropic.com`.
- Robot emoji: `🤖`.
- `Generated with [Claude Code]` (verbatim, case-insensitive).
- `claude.com/code` (verbatim).
- `AI-generated` (verbatim, case-insensitive).
- Attribution **phrases** (case-insensitive verb, model name token):
  - `Generated by (Claude|Anthropic|Sonnet|Opus|Haiku)`
  - `Authored by (Claude|Anthropic|Sonnet|Opus|Haiku)`
  - `Written by (Claude|Anthropic|Sonnet|Opus|Haiku)`
  - `Co-?authored by (Claude|Anthropic|Sonnet|Opus|Haiku)`
  - `by Claude`, `by Anthropic` as standalone trailing attribution (preceded by
    whitespace/punct, end-of-line or punct after) — narrow to avoid "stand by Claude's
    decision" style false positives by requiring the phrase appear at end-of-line or
    immediately before `.`/`,`/`;`. (See task T2 detail for regex.)

**Allow (do not block):**
- Bare technical mentions of `Claude` / `Anthropic` / `Sonnet` / `Opus` / `Haiku` outside
  attribution phrasings. Examples that must pass:
  - `Viktor (sonnet-high) implemented the worktree guard`
  - `chore: update agent def for opus model selection`
  - `Promote plan via Orianna; routes through Sonnet executors`
  - `Anthropic's tool-use API` (technical reference, no attribution verb)

## Override removal

Strip `Human-Verified: yes` handling entirely from:
- `scripts/hooks/commit-msg-no-ai-coauthor.sh` — delete the early-exit block at lines
  43–46 and the trailing `If a human collaborator…` rejection-message paragraph
  (lines 122–124).
- `scripts/ci/pr-lint-no-ai-attribution.sh` — delete the early-exit block at lines
  30–33 and the trailing rejection-message paragraph (line 78–79).
- `.claude/agents/_shared/no-ai-attribution.md` — rewrite line 3 to remove the override
  reference. Replace `Legitimate human pair-programming uses the 'Human-Verified: yes'
  override trailer instead.` with: `No override mechanism — if you need the trailer for
  legitimate authorship, omit attribution entirely.`
- `CLAUDE.md` Rule 21 — delete the final sentence
  (`Override for all three layers: 'Human-Verified: yes' trailer in the commit message
  or PR body.`).

History is not rewritten. Orphan `Human-Verified:` lines in past commits / PR bodies
become inert text.

### T2. Tighten detector + remove override (impl)

- kind: impl
- estimate_minutes: 25
- files:
  - `scripts/hooks/commit-msg-no-ai-coauthor.sh`
  - `scripts/ci/pr-lint-no-ai-attribution.sh`
- detail:
  1. **Remove `Human-Verified: yes` early-exit blocks** in both scripts (4 lines each
     plus the rejection-message hint paragraph).
  2. **Replace** the `BODY_MARKERS` bare-token regex with attribution-phrase regexes.
     Keep these as universal blocks (no context required):
     - Pattern A: `^Co-Authored-By:` (case-insensitive) — unchanged.
     - Pattern B: AI email domains — unchanged.
     - Pattern C-verbatim: `🤖`, `Generated with \[Claude Code\]`, `claude\.com/code`,
       `AI-generated` (the latter case-insensitive). Note: tighten `claude.com` →
       `claude\.com/code` so that `claude.com` URLs in unrelated docs do not false-
       positive (current script blocks bare `claude.com`).
     - Pattern C-attribution-phrase (NEW, replaces bare-token C): case-insensitive
       verb, exact-token model name. ERE example (BSD/GNU compatible):
       ```
       (Generated|Authored|Written|Co-authored|Coauthored)[[:space:]]+by[[:space:]]+(Claude|Anthropic|Sonnet|Opus|Haiku)
       ```
       Plus a narrow trailing-attribution form:
       ```
       (^|[[:space:]])by[[:space:]]+(Claude|Anthropic)([[:space:]]*[.,;]|[[:space:]]*$)
       ```
       The verb-phrase regex is case-insensitive (`grep -iE`); the model-name
       capture group accepts any case (`Claude`/`claude`/`CLAUDE` all match).
  3. Keep the dedup + rejection-message machinery; **update the rejection message** in
     both scripts to remove the "If a human collaborator…override" paragraph and to
     reflect that bare technical mentions are allowed.
- DoD: Both scripts pass shellcheck; T1 cases that were xfail now pass; bare-mention
  pass cases (`Viktor (sonnet-high) implemented…`) do not block. CI / commit-msg
  rejection messages no longer reference `Human-Verified: yes`.

## QA Plan

Tests live in the existing shell-harness style (`tests/hooks/test_commit_msg_no_ai_coauthor.sh`,
`tests/ci/test_pr_lint_no_ai_attribution.sh`) — both already extant from the
2026-04-25 defense-in-depth plan. We extend, not replace.

**Pass cases (must NOT block) — exercise both scripts:**
1. Body: `Viktor (sonnet-high) implemented the worktree guard.`
2. Subject: `chore: update agent def for opus model selection`
3. Body: `Promote plan via Orianna; routes through Sonnet executors.`
4. Body: `Anthropic's tool-use API supports streaming.`
5. Body: `The Claude CLI prompts for confirmation.` (bare mention, no attribution verb)
6. Body containing **only** `Human-Verified: yes` plus a clean message — passes because
   the trailer is now inert text and there is no attribution marker. (Asserts the
   trailer is not toxic on its own — it's just ignored.)

**Fail cases (MUST block):**
1. Trailer: `Co-Authored-By: Claude Opus 4.7 <noreply@anthropic.com>`
2. Trailer: `Co-Authored-By: Anybody Else <x@y.test>` (any Co-Authored-By, even non-AI)
3. Body: `🤖 Generated with [Claude Code]`
4. Body containing `https://claude.com/code` URL
5. Body: `Generated by Claude` (attribution phrase)
6. Body: `Authored by Sonnet` (attribution phrase, model name)
7. Body: `Written by Anthropic`
8. Body: `Co-authored by Opus`
9. Body: `Implemented by Claude.` (trailing-attribution form)
10. Email-domain match: any line containing `<x@anthropic.com>`

**Override-no-op cases (the critical regression-prevention assertion):**
11. Commit body containing `Generated by Claude` AND `Human-Verified: yes` →
    **still blocks**. (Asserts the override is gone.)
12. PR body containing `Co-Authored-By: Foo <a@b>` AND `Human-Verified: yes` →
    **still blocks**.

**Invariants protected:**
- Bare technical mentions of model names pass on their own merit (no override needed).
- The override trailer is inert: presence does not unblock; absence does not block.
- Attribution **context** (verb + name) is the trigger, not bare-string presence.
