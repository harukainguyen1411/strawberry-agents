# P1.2-A — TDD plan for `scripts/deploy/_lib.sh`

## 1. Scope

**In scope:** Behavioural tests pinning each exported function in `scripts/deploy/_lib.sh`, plus two static gates (shellcheck clean, grep-for-bare-`firebase deploy`).

**Out of scope:**
- Surface scripts (`scripts/deploy/deploy-functions.sh`, etc.) — separate tasks (P1.8+).
- Test harness scripts (`scripts/test-functions.sh` etc.) — P1.4–P1.7.
- Integration with a real Firebase project (P1.12 dry-run covers that).
- Audit-log schema validation (covered by dashboard consumer in P2.x).

## 2. Function contracts under test

From `plans/in-progress/2026-04-17-deployment-pipeline-tasks.md:103-116` and ADR §1a.7, §5, §8:

| # | Function | Contract summary |
|---|---|---|
| 1 | `dl_require_clean_tree` | Exits non-zero if `git status --porcelain` non-empty. Rule 1 (no uncommitted work). |
| 2 | `dl_require_main_or_branch_flag` | Exits non-zero if on non-main branch without an explicit `--branch` opt-in. |
| 3 | `dl_decrypt_env <project>` | Invokes `tools/decrypt.sh` for each cipher in `secrets/env/<project>.env.age`; never calls raw `age -d`; never prints plaintext to stdout/stderr. |
| 4 | `dl_audit_log_start` | Appends `status: started` JSONL record to `logs/deploy-audit.jsonl` with required fields (project, surface, ref, timestamp, invoker). |
| 5 | `dl_audit_log_finish` | Appends `status: success\|failure` JSONL record, duration_ms, matches the prior `started` record's id. |
| 6 | `dl_require_firebase_only_flag` | Rejects a deploy invocation string that lacks `--only <surface>`. ADR §1a.7 hard contract. |
| 7 | `dl_detect_firebase_auth` | Echoes `sa-file` when `GOOGLE_APPLICATION_CREDENTIALS` is set and file exists; else `personal-cli` when a Firebase CLI token is present; else exits non-zero. |

Plus two static gates:
- **G1. `shellcheck scripts/deploy/_lib.sh`** — zero findings at default severity.
- **G2. Grep-for-bare-deploy** — repo-wide grep fails if any script under `scripts/deploy/**` contains `firebase deploy` without `--only` on the same logical line (allow continuations).

## 3. Framework, layout, and conventions

- **Framework:** `bats-core` (already the norm for bash tests in this repo; no new dep if already pinned — otherwise Vi's P1.2-B task installs via `scripts/install-bats.sh` or pins a vendored submodule).
- **Test file:** `scripts/deploy/__tests__/_lib.bats` (co-located with the lib under a `__tests__/` dir — mirrors `apps/myapps/functions/__tests__/` convention from P1.4).
- **Fixture dir:** `scripts/deploy/__tests__/fixtures/` — holds fake `tools/decrypt.sh` stubs, fake `git` stubs, synthetic `logs/` scratch, dummy `.env.age` blobs.
- **Static gates:** one-liner bash test scripts in `scripts/deploy/__tests__/` — `check-shellcheck.sh` and `check-no-bare-deploy.sh`. Both invoked from bats as final gates.
- **Runner:** `scripts/test-deploy-lib.sh` (Vi will author under P1.5 companion task; the bats suite is callable standalone with `bats scripts/deploy/__tests__/_lib.bats`).
- **POSIX discipline (Rule 10):** tests use only features available in bash 3.2 (Git Bash on Windows baseline). No `declare -A`, no `mapfile`, no `<<<` where a heredoc works.

## 4. Failure modes to pin (the "what could go wrong")

Each test in §5 maps to at least one of these. Before tests are written, Jayce / Vi read this list and confirm none are missing:

- **FM-1.** `_lib.sh` invoked from a dirty working tree and silently proceeds — deploy uses stale/uncommitted code.
- **FM-2.** Deploy runs from a feature branch without operator consent — wrong code shipped.
- **FM-3.** Env decrypt calls raw `age -d` or pipes plaintext through an argv — violates Rule 6.
- **FM-4.** Env decrypt echoes the plaintext secret to stdout/stderr — leaks to terminal/CI log.
- **FM-5.** Audit log start/finish mismatch — orphan `started` with no `finish`, or `finish` without matching id.
- **FM-6.** Audit log writes to the wrong path or malformed JSON — dashboard consumer breaks.
- **FM-7.** `firebase deploy` invoked without `--only` — cross-surface blast radius (ADR §1a.7).
- **FM-8.** Auth detection silently falls back to personal CLI when SA file path is set but file is missing — prod deploy uses wrong identity.
- **FM-9.** Auth detection exits 0 when no auth is present — deploy proceeds unauthenticated and later fails opaquely.
- **FM-10.** `_lib.sh` contains zsh-isms (`[[ =~ ]]` quirks, `print -r`, etc.) — fails on Git Bash.
- **FM-11.** `_lib.sh` uses non-portable `readlink -f`, GNU-only `date --rfc-3339`, etc. — fails on BSD/macOS bash.
- **FM-12.** Caller sources `_lib.sh` with `set -e` and a helper's internal `grep` nonzero-exit aborts the caller — helpers must be safe-to-source.

## 5. xfail catalog

Format per entry:
- **ID / name / function under test**
- **Assertion** (what the test checks)
- **File path** (which bats file / test block)
- **Failure mode pinned** (FM-# from §4)
- **Initial expected result** (must fail against empty/absent `_lib.sh` — xfail-first)
- **Post-impl expected result** (must pass once Jayce's `_lib.sh` is in)

All tests live in `scripts/deploy/__tests__/_lib.bats` unless noted. Each test's bats description begins with `xfail-P1.2:` so the xfail-enforcement check (TDD plan rule 1) can trace it.

## 6. Test isolation / hermeticity

Every bats block must:
- `setup()`: create a throwaway `$BATS_TEST_TMPDIR/repo` with `git init`, an empty `logs/` dir, a `secrets/env/` containing a fixture `.env.age`, and a `PATH` prepended with `$BATS_TEST_TMPDIR/bin` to front-load stubs.
- `teardown()`: clean up via bats' automatic tmpdir removal. No file under the real repo's `logs/`, `secrets/`, or working tree may be touched.
- Stubs for `git`, `firebase`, `age`, `tools/decrypt.sh` live in `scripts/deploy/__tests__/fixtures/bin/` and are copied into `$BATS_TEST_TMPDIR/bin` per test.
- No test calls the real `age`, real `firebase`, real network, or modifies the real working tree. The pre-commit hook's dirty-tree check must not trip during `git commit` of this plan.
