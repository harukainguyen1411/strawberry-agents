# Retired: v2-plan-structure-lint

Archived 2026-04-24. Both plan-structure lint hooks — `pre-commit-zz-plan-structure.sh` (the heavier 559-line linter from 2026-04-21) and the older `pre-commit-t-plan-structure.sh` (the 228-line linter from the 2026-04-20 plan-structure-prelint plan) plus its test harness `test-pre-commit-plan-structure.sh` — were retired because they duplicate scope that the Orianna v2 Opus agent gate explicitly declines to enforce. Orianna v2 operates with a substance-over-format discipline: it does not block on purely structural "tiny bits" such as bare-filename backtick path references, canonical heading format, or forward self-references in plan prose. The mechanical pre-commit lints produced false positives on non-path backtick tokens in plan prose (verified 2026-04-23 and 2026-04-24). An Explore audit initially identified only the `-zz-` variant as redundant; the `-t-` variant surfaced when it re-fired after the first archival and was archived in the same session. Structural validation of plan files now lives entirely with the Orianna agent gate (see `plans/implemented/personal/2026-04-23-orianna-gate-simplification.md`). The 15 other pre-commit hooks remain active (secrets, TDD, identity-anonymity on commits, artifact, staged-scope, commit-msg, etc.) — all orthogonal to Orianna's scope.
