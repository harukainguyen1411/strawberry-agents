# Retrospection Dashboard

Plan-centric observability dashboard built from Claude JSONL transcripts, subagent sentinels,
and git plan-lifecycle history. Static HTML output, DuckDB-over-JSONL queries, zero managed
database.

Plan-Ref: plans/approved/personal/2026-04-25-retrospection-dashboard-and-canonical-v1.md

## Four-step pipeline

```
ingest → events.jsonl → query → render
```

1. **ingest** — scans four upstream sources, writes `~/.claude/strawberry-usage-cache/events.jsonl`
2. **events.jsonl** — append-only canonical event log (turn, dispatch, plan-stage)
3. **query** — DuckDB SQL files in `queries/*.sql` run against events.jsonl
4. **render** — writes `dist/data/*.json` + static HTML to `dist/`

## Directory tree

```
tools/retro/
  ingest.mjs                  # Source scanner (§Q1 four upstream sources)
  render.mjs                  # DuckDB query runner + HTML generator
  lib/
    sources.mjs               # Per-source readers (parent JSONL, subagent JSONL+meta, sentinels, git log)
    plan-stage-detect.mjs     # Three-signal layered plan-stage detection (§Q2)
    duckdb-runner.mjs         # DuckDB CLI wrapper (read_ndjson_auto)
    html-generator.mjs        # Static HTML builder (index + plan-detail pages)
    template.mjs              # Zero-dep {{TOKEN}} interpolator + escapeHtml
    mtime-cache.mjs           # Per-source mtime cache for incremental re-scan (Phase 2)
  queries/
    plan-rollup.sql           # Per-plan token cost + wall-active rollup
    coordinator-weekly-skeleton.sql  # Inline vs delegate discrimination (§Q8)
    *.expected.json           # Golden-file fixtures for TP1.T5
  templates/
    index.html.tpl            # Dashboard index template
    plan-detail.html.tpl      # Per-plan detail template
  fixtures/                   # Test fixtures
  __tests__/                  # Node test suites (node --test) + bats e2e
  dist/                       # Emitted static site — gitignored, regenerated by retro:render
    data/
      .gitkeep
```

## Prerequisites

- **Node.js >= 18** (standard stdlib only; no npm install required)
- **DuckDB CLI** — `duckdb` must be on `PATH`. Install via:
  - macOS: `brew install duckdb`
  - Linux: download from https://duckdb.org/docs/installation (CLI binary)
  - The DuckDB CLI is used by `lib/duckdb-runner.mjs` to run `.sql` queries against
    `events.jsonl` via `read_ndjson_auto`. It is NOT included as an npm dependency —
    the tool intentionally avoids managed database infrastructure.
- **git** — required by `ingest.mjs` when reading live git log (not needed for test fixtures)
- **bats** — optional, only required to run the e2e suite in `__tests__/e2e-pipeline.bats`

## events.jsonl output location

`ingest.mjs` writes `events.jsonl` to:
- `<--cache-dir>/events.jsonl` when `--cache-dir` is supplied (test isolation)
- `~/.claude/strawberry-usage-cache/events.jsonl` by default (production)

This file is the canonical event log consumed by `render.mjs`. It is a full rebuild on every run
(no incremental append in Phase 1).

## Running locally

```sh
# Ingest from live sources
node tools/retro/ingest.mjs

# Render (uses default events.jsonl location)
node tools/retro/render.mjs \
  --events ~/.claude/strawberry-usage-cache/events.jsonl \
  --queries-dir tools/retro/queries \
  --out-dir tools/retro/dist

# Open dashboard
open tools/retro/dist/index.html
```

## Time normalization

Wall-clock `wall_active_delta_s` is computed per assistant turn as the delta between consecutive
assistant timestamps. Gaps greater than 90 seconds between turns are treated as idle and stripped
to zero (not counted toward `wall_active_minutes`). A gap of exactly 90 seconds is not stripped
(≤90s criterion — boundary is inclusive active).

## Plan-stage detection (§Q2)

Three signals in priority order:

1. **Orianna trailer** (`Promoted-By: Orianna` in git commit) — canonical
2. **Frontmatter mtime** (`status:` change in plan file detected via git log) — corroborative
3. **Dispatch-prompt slug match** (plan path cited in first 200 chars of dispatch) — fallback

When trailer and frontmatter disagree on stage, trailer wins and `signal_conflict: 'frontmatter-newer-than-trailer'` is logged (OQ-R3 ruling, Swain 2026-04-25).
