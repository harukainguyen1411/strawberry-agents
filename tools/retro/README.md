# Retrospection Dashboard

Plan-centric observability dashboard built from Claude JSONL transcripts, subagent sentinels,
and git plan-lifecycle history. Static HTML output, DuckDB-over-JSONL queries, zero managed
database.

Plan-Ref: plans/approved/personal/2026-04-25-retrospection-dashboard-and-canonical-v1.md

## Four-step pipeline

```
ingest → events.jsonl → query → render
```

1. **ingest** — scans four upstream sources, writes `~/.claude/strawberry-usage-cache/events.jsonl`
2. **events.jsonl** — append-only canonical event log (turn, dispatch, plan-stage)
3. **query** — DuckDB SQL files in `queries/*.sql` run against events.jsonl
4. **render** — writes `dist/data/*.json` + static HTML to `dist/`


## Running locally

```sh
# Ingest from live sources
node tools/retro/ingest.mjs

# Render (uses default events.jsonl location)
node tools/retro/render.mjs \
  --events ~/.claude/strawberry-usage-cache/events.jsonl \
  --queries-dir tools/retro/queries \
  --out-dir tools/retro/dist

# Open dashboard
open tools/retro/dist/index.html
```

## Time normalization

Wall-clock `wall_active_delta_s` is computed per assistant turn as the delta between consecutive
assistant timestamps. Gaps greater than 90 seconds between turns are treated as idle and stripped
to zero (not counted toward `wall_active_minutes`). A gap of exactly 90 seconds is not stripped
(≤90s criterion — boundary is inclusive active).

## Plan-stage detection (§Q2)

Three signals in priority order:

1. **Orianna trailer** (`Promoted-By: Orianna` in git commit) — canonical
2. **Frontmatter mtime** (`status:` change in plan file detected via git log) — corroborative
3. **Dispatch-prompt slug match** (plan path cited in first 200 chars of dispatch) — fallback

When trailer and frontmatter disagree on stage, trailer wins and `signal_conflict: 'frontmatter-newer-than-trailer'` is logged (OQ-R3 ruling, Swain 2026-04-25).
