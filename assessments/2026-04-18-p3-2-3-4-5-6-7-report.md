## P3.6 — BEE_SISTER_UIDS status

**References found:**

- `apps/myapps/functions/src/beeIntake.ts` — `defineString("BEE_SISTER_UIDS", { default: "" })`
- `apps/myapps/functions/src/index.ts` — `defineString("BEE_SISTER_UIDS", { default: "" })`
- `apps/myapps/functions/README.md` — documents it as a local `.env` override for emulator auth bypass

**Classification:** This is a **Firebase Functions parameter** (set via `firebase functions:config:set` or `.env` file for the emulator), NOT a GitHub Actions secret. It does not appear in any `.github/workflows/*.yml` file, in `secrets/encrypted/`, or as a `gh secret`.

**Provisioning recommendation:** Duong does not need to set `BEE_SISTER_UIDS` as a GitHub secret. It should be set as a Firebase Functions environment variable via the Firebase Console (Functions → Configuration → Add variable: `BEE_SISTER_UIDS=<comma-separated UIDs>`) or in the `.env` file for local emulator use. The `default: ""` means functions will run with an empty allowlist if not set — effectively disabling Bee authentication — which is acceptable for the emulator but must be configured in production.
