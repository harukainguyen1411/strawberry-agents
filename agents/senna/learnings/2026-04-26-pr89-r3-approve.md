## What changed since round 2

The round-2 finding was: `coordinator IS NOT NULL` filters all real-production turns
because turn events lacked the field. Round 3 plumbs coordinator into turn events at
ingest, on BOTH parse paths:

- `parseSubagentSession` — `coordinator: meta ? (meta.coordinator || null) : null` on every
  delegated `kind:'turn'` event. Trivially mirrors the C1 dispatch-event field.
- `parseParentSession` — reads sibling `<sessionId>.meta.json` via `existsSync` + try/catch
  `JSON.parse`, derives `coordinator` once at session scope, plumbs onto every inline turn.
- Orphan-sentinel branch — explicit `coordinator: null, ts: null` with TODO comment.

Backward-compat is correct: sessions without a meta sidecar get `coordinator: null` and are
simply absent from coordinator-weekly (filtered by `coordinator IS NOT NULL`). No fabrication.

## What unstuck the verdict

The new round-trip regression test is exactly what round-2 demanded:

1. Builds source-shape fixture (`<sess>.jsonl + <sess>.meta.json + subagents/<agent>.{jsonl,meta.json}`) in tmpdir.
2. Invokes real ingest via `execSync('node ingest.mjs --cache-dir ${tmp}')`.
3. **Pre-asserts** every inline + delegated turn in the emitted `events.jsonl` carries
   `coordinator='evelynn'` BEFORE running SQL. This catches parser drift independent of
   downstream SQL behavior — a future regression that drops the field on either parse path
   fails with a precise message naming the broken path.
4. Runs the actual SQL against ingest-produced events.jsonl, asserts non-null ratio +
   non-degenerate counts.

This is the structural shape the round-2 learning specified ("regression tests must
round-trip ingest, not pre-bake fixtures"). It would have failed against round-2 head.

## Lesson reinforced — pre-assertion on parser output is gold

The round-trip test puts an `assert.strictEqual(t.coordinator, 'evelynn', ...)` on every
turn event read from the real ingest output, BEFORE the SQL even runs. This decouples
parser-drift detection from SQL-behavior detection. If a future change accidentally drops
`coordinator` on parseParentSession but leaves parseSubagentSession intact, only inline
counts go to zero — the SQL ratio assertion would catch it as "non-zero but skewed", but
the per-turn pre-assertion immediately names the broken path. **Cheap insurance against
future single-path regressions in compound-predicate land.**

## Process note — three rounds, same architecture

Round 1 found fixture-vs-prod drift on `kind`. Round 2 found the same drift class on
`coordinator`. Round 3 fixes both at the source-of-truth layer (the parsers themselves)
plus a round-trip test that catches the entire predicate composition. Three rounds is
acceptable cost for this PR's scope; the lesson is that reviewing **compound** WHERE
clauses requires per-predicate fixture-vs-prod audit on round 1, not iterative discovery.
