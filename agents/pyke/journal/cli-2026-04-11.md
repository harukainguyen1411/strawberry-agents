# Pyke — 2026-04-11

Short research session today. Duong asked whether a hook could enforce `/end-subagent-session` on every subagent close — a real gap, since agents run as background tasks and silently exit without persisting memory.

I looked into Claude Code's hook events. `SubagentStop` exists but fires post-exit — it can't intercept, only detect. No `PreSubagentStop` exists today, so a hard gate at the harness level isn't possible without Anthropic adding it upstream.

The best available option is a sentinel-file pattern: the skill touches a file on clean close; `SubagentStop` checks for it and warns Evelynn if absent. Post-hoc enforcement, not prevention — but better than nothing.

Wrote the plan to `plans/proposed/2026-04-11-subagent-stop-hook.md`. Left two open questions for Duong: whether a warning is sufficient or a hard gate is needed, and that the `SubagentStop` stdin schema needs empirical testing before implementation.

The underlying problem is real: planners like me currently lose all session context when run as background tasks. The sentinel approach is a step toward closing that gap.
