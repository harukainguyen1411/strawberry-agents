# P3.9 Smoke Test — strawberry-app Workflow Validation

**Date:** 2026-04-18

## What happened

Ran a smoke test PR against `harukainguyen1411/strawberry-app` after Phase 3 migration. Pushed a trivial version bump to `chore/smoke-test-migration`, opened PR #18, waited for all 10 triggered workflow runs to settle.

## Key findings

- 10 green, 1 red (`Preview` workflow — pre-existing issue, not migration regression)
- `Preview` fails because `scripts/composite-deploy.sh` errors when no `apps/*/dist` exists — it runs before any build step on package.json-only changes
- The actual Firebase Hosting PR Preview (separate workflow, run 24605781415) passed
- Branch protection, secrets, lint-slugs regression guard, TDD Gate, CI, E2E all green
- `qa.yml` did not trigger — likely not PR-triggered or requires UI changes

## Lessons

- When a repo has multiple Firebase-adjacent workflows (`Preview` + `Firebase Hosting PR Preview`), they may behave differently. The composite-deploy wrapper needs a "no dist = skip" guard.
- On package.json-only PRs, E2E and Playwright workflows still trigger but fast-exit (8s) — good path filtering.
- `gh run list --branch <branch>` is the right command for watching PR runs; the `--json` flag with `jq` to check `status != "completed"` works well in until-loops.
