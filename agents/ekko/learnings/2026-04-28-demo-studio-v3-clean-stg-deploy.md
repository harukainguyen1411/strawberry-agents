## Quirks encountered

1. **Dirty tree from test artifacts**: `test-results.json` and `test-run-history.json` are generated by `conftest_results_plugin.py` on every pytest run. They were untracked and not gitignored, causing `deploy.sh` dirty-check to fail. Fixed by adding them to `.gitignore` (commit `ac64112`). This is a one-time fix for this branch; the pattern should be applied to future branches with `conftest_results_plugin`.

2. **Traffic pinning**: The service had an explicit pinned traffic config (`revisionName: demo-studio-00044-p68, percent: 100`). After `gcloud run deploy --source`, Cloud Run created revision `00045-x9j` but left it as "Retired" (0% traffic) because traffic was pinned. Had to explicitly flip with `gcloud run services update-traffic ... --to-revisions=demo-studio-00045-x9j=100`.

3. **Revision timestamp vs deploy start**: Cloud Build finish time (04:23:37Z) appeared earlier in the revision list than expected because local deploy clock vs GCP timestamps differ. The BUILD_SHA env var is the reliable way to confirm which revision corresponds to which source commit.

4. **No FORCE_DIRTY needed** after gitignore fix — clean deploy proceeded normally.
