# PR #25 / #26 / #28 CI Fixes

Date: 2026-04-19

## PR #25 — Firebase Hosting PR Preview "No currently active project"

Root cause: `preview.yml` copied `apps/myapps/firebase.json` to repo root (entryPoint: .)
but NOT `.firebaserc`. Without `.firebaserc` at root and `vars.FIREBASE_PROJECT_ID` unset,
firebase-tools had no project reference.

Fix: added `cp apps/myapps/.firebaserc .firebaserc` to the "Copy firebase.json to repo root"
step (renamed step accordingly).

Commit: a303dd6

## PR #26 — npm ci lockfile drift (@vitest/coverage-v8)

Root cause: root `package-lock.json` had `@vitest/coverage-v8@4.1.4` for `apps/myapps`
workspace, but `apps/myapps/package.json` pins it to `4.0.18`. Caused `npm ci` to fail
at repo root with "lock file's @vitest/coverage-v8@4.1.4 does not satisfy @vitest/coverage-v8@4.0.18".

Fix: ran `npm install` at repo root in the branch worktree, committed the regenerated
`package-lock.json`.

Commit: 99841bc

## PR #28 — "No portal dist found" in composite-deploy.sh

Root cause: `turbo run build --filter=...[origin/main]` skips `apps/myapps` when the PR
doesn't touch myapps source files. `composite-deploy.sh` then exits 1 because
`apps/myapps/dist` is absent.

Fix: added an explicit `npm run build` step in `apps/myapps` BEFORE `composite-deploy.sh`
in `preview.yml`. Scripts shouldn't silently trigger builds, so the build step stays in
the workflow.

Commit: e9650bb (merge commit: 5aaa1a5)

## Pattern

`preview.yml` runs for all PRs but was missing two invariants:
1. `.firebaserc` must accompany `firebase.json` at the entryPoint root
2. `apps/myapps/dist` must exist before `composite-deploy.sh` runs, even for non-frontend PRs
