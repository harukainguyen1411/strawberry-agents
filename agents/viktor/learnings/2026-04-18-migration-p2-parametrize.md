# Learnings: Migration Phase 2 — Repo Slug Parametrization

## Firebase `defineString` default gotcha

`defineString("BEE_GITHUB_REPO", { default: "..." })` — the `default` field is evaluated at deploy-config time, not at runtime. Passing `process.env.GITHUB_REPOSITORY` as the default would read it during the module load in the deployment environment, which may be a different environment than runtime. Safe pattern: use `default: ""` and add runtime fallback: `paramValue.value() || process.env.GITHUB_REPOSITORY || ""`. This keeps the Firebase param override path intact.

## Shell script slug parametrization pattern

Three-tier fallback for shell scripts that need a repo slug:
1. `$1` positional arg (explicit invocation)
2. `$GITHUB_REPOSITORY` env var (GitHub Actions implicit)
3. `git remote get-url origin | sed ...` parsing

Helper function to parse git remote URL handles both HTTPS and SSH forms:
```bash
_derive_repo_from_remote() {
  local remote_url
  remote_url="$(git remote get-url origin 2>/dev/null)" || return 1
  echo "$remote_url" | sed -E 's|.*github\.com[:/]||; s|\.git$||'
}
```

## Regression guard hook dispatch

The strawberry-managed pre-commit dispatcher auto-invokes `pre-commit-*.sh` files. For a guard named `check-no-hardcoded-slugs.sh` (without the prefix, for standalone usability), the cleanest wiring is a thin `pre-commit-check-no-hardcoded-slugs.sh` wrapper generated by `install-hooks.sh`. The `install-hooks.sh` creates the wrapper only if absent (idempotent).

## POSIX-portable grep for multi-extension scan

Avoid `rg` or extended grep for portability. Use grep's `--include` flag with multiple calls or in a loop. In the hook: build `INCLUDE_FLAGS` by iterating over extensions. The guard exits cleanly even when `grep` returns exit 1 (no matches) by appending `|| true`.

## Allowlist path matching in pure shell

For a glob-based allowlist (one entry per line, `**` suffix meaning "any file under that directory"):
```bash
case "$file_rel" in
  "$dir_prefix/"*) return 0 ;;
esac
```
Use `case` for pattern matching — POSIX portable, no `[[ ]]` bashism required.

## git update-index --chmod=+x workflow

`git update-index --chmod=+x <path>` only works after the file is already tracked. Must `git add` the file first, then run `--chmod=+x`. Running it before `git add` gives "cannot add to the index - missing --add option" error.
