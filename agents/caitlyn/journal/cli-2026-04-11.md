# Caitlyn — Session Journal — 2026-04-11

Invoked by Evelynn to investigate a blank-page issue on the MyApps production deployment at https://myapps-b31ea.web.app. No plan file was referenced — this was a pure diagnostic task, not an implementation run.

The investigation moved quickly. Playwright confirmed the page was fully blank, and a single console error was the entire evidence trail: a hard throw from `firebase/config.ts` stating that VITE_FIREBASE_ environment variables were missing. The throw occurs at module load time, before Vue ever mounts, which is why the page is completely empty — no partial render, no error boundary, just silence.

Tracing the root cause upstream: Vite bakes `import.meta.env.*` at build time. The GitHub Actions deploy workflow (`myapps-prod-deploy.yml`) runs `npm run build` with no `env:` block injecting the Firebase secrets. So the build produces a bundle where every `VITE_FIREBASE_*` reference resolves to `undefined`, which trips the guard in `config.ts`.

The triggering commit was almost certainly `9ce763e` — "bee firebase rewrite" — which introduced `src/firebase/config.ts` with that hard throw. Before that commit the guard may not have existed. Once a subsequent push to main touched `apps/myapps/**`, the deploy workflow fired, built without the vars, and shipped a broken bundle.

The fix is straightforward: add the VITE_FIREBASE_* secrets to GitHub and inject them into the Build step's `env:` block. I documented it clearly in my report but did not touch any code — this was a diagnosis-only assignment.

Decent session. Clean evidence, minimal tooling overhead, clear conclusion.
