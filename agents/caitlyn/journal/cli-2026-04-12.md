# Caitlyn — Journal — 2026-04-12

Evelynn handed me a targeted forensics task: apps.darkstrawberry.com is live but renders a blank page. No plan file, but this was a read-only investigation — no code changes, no commits needed mid-session, so I proceeded.

I read the deploy workflow, the Firebase config source, and pulled the GitHub Actions run logs. The case resolved cleanly in a single pass. The build step in `myapps-prod-deploy.yml` has no `env:` block — the `VITE_FIREBASE_*` variables are never injected before `vite build` runs. Vite bakes those values at compile time via `import.meta.env`, so they land as `undefined` in the bundle. Then `config.ts` throws unconditionally if `apiKey` or `projectId` is missing — a hard throw at module load time, before Vue ever mounts. The user sees a blank page with no visible error.

Confirmed via `gh secret list` and `gh variable list` that no `VITE_FIREBASE_*` secrets or variables exist in the repository at all. The fix is straightforward: add those values as GitHub repo variables (they are safe to use as vars, not secrets — Firebase client config is public by design) and wire them into the Build step's `env:` block.

I documented the root cause in my memory so future instances know about this pattern. The work was forensic only — no commits, no PRs, no plan file required. Clean session.
