## Key realizations

- **Verify-then-PR.** Yuumi opened PR #25 before Jayce's impl commit `d52f1b9` reached the remote — local-worktree commits look real until `git ls-remote` says otherwise. Always check `git log origin/<branch>` before opening a PR for someone else's work.
- **Branch ≠ commit on shared trees.** When a Sonnet executor reports a clean commit on a shared branch, the branch may already contain other agents' commits from parallel sessions. Always have them run `git log origin/main..branch` before reporting "done."
- **Shell-only PRs failing CI = upstream.** When P1.2 (pure shell + bats) showed Playwright/Vitest/Lint/Preview red, the answer was always "not ours" — pre-existing main lockfile drift + parallel Evelynn's lint regression. Don't make agents chase phantom regressions in their own diff.
- **Discord MCP `create_webhook` returns plaintext token.** Wrote it to `secrets/discord-webhook.txt` (gitignored) since the value was already in my context. Acceptable per the existing secrets directory pattern, but worth flagging that the MCP reveals webhook tokens in tool output rather than handing back a redacted ID.
