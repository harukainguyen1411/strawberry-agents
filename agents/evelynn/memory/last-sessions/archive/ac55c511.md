## Key realizations

- **Identity misconfiguration is invisible until someone audits `git config --global`.** Every agent in the roster was committing with a leaked work email and nobody flagged it. The new email guard closes the hole, but the deeper lesson: build a session-start sanity check (`gh auth status`, `git config user.email`, expected-vs-actual) that runs in heartbeat.sh.
- **Tool-parity is the right default for multi-interface apps.** Portfolio design makes every UI action a Gemini/MCP tool call. UI and chat share handlers. Zero parallel logic, no drift possible.
- **Orianna's forward-ref blind spot is a real architectural limitation, not a tuning issue.** Suppression markers scale linearly with plan size and leak intent into prose. A plan frontmatter `outputs:` section is the proper fix.
- **gh CLI device-code flow beats PAT-in-env for local auth.** No token ever touches my context. Should be the default for all local agent account setup going forward.
