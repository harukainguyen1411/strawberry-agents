## Key realizations

- **Subagent MCP auth doesn't share with parent.** Each spawn has its own MCP server process with its own auth state. OAuth at the top level does not authenticate subagents. Per-spawn OAuth is required. AND the MCP client does not re-query tools/list after auth lands mid-spawn — the tool roster snapshots at spawn-start. So even with auth, post-auth tools (use_figma, etc.) don't materialise. The harness-level fix isn't available; either a fresh spawn or top-level execution is the only path.
- **Agent-def `tools:` field caches at session start, not per-spawn.** Editing `.claude/agents/<name>.md` mid-session has zero effect on subsequent spawns from that same Evelynn session — restart required to flush. The S40 cache rule extends to tools, not just model.
- **Removing `tools:` field entirely makes an agent inherit all tools** — same pattern as Akali's def. This is the cleanest way to grant broad MCP access to a roster agent without enumerating every plugin tool. Trade-off vs least-privilege.
- **Figma Starter plan caps:** 3 pages per file, MCP tool-call rate limit hit silently after ~5 use_figma calls. Free-tier ceiling is real; document expected output before invoking.
- **TeamCreate vs background Agent:** the team task-list view is genuinely useful for visualizing parallel work, but the spawn ceremony + shutdown_request dance + per-spawn MCP auth issues meant Duong dissolved mid-session. Background Agents handled the rest cleanly. For phase-gated work where coordination is mostly Evelynn → 1-3 specialists per phase, background Agents win on simplicity.
- **Subagent context death is the new dominant failure mode.** Three different agents (Seraphine x2, Viktor x1) died on context overflow / stream idle timeouts. The mitigation: tighter scopes per spawn (4 tasks max for builders), pre-baked answers for tool/library research questions, and explicit "STOP after task X" instructions so the agent doesn't grind into a context wall trying to do "everything possible."
- **Pre-existing tech debt in sibling apps will sweep into "affected" turbo builds.** task-list and read-tracker had silent lint errors that only surfaced when portfolio scaffolding made the lint task active. Allow time for this kind of fallout when adding any new sibling under apps/myapps/.
