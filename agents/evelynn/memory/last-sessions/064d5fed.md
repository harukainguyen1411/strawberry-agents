## Key realizations

- **Every agent auth'd as one account = Rule 18 structurally unsatisfiable.** Fix is a second identity scoped to reviewer role only. Smoke test proves it. This was the unsung-hero problem of the last month.
- **Auto-rebase workflows are a CI anti-pattern** when combined with many-open-PR state. O(N) retriggers per merge. On-demand update-branch on the single next-in-line PR scales to any N.
- **plan-promote.sh is proposed-only** (confirmed twice this session). For approved→in-progress or in-progress→implemented, `git mv` + frontmatter rewrite is the sanctioned pattern.
- **Orianna fact-check gate blocks on forward-looking file references.** Suppress with `<!-- orianna: ok -->` inline when the plan's §3 is by definition talking about files-to-be-created.
- **Firebase CLI picks up `.firebaserc` from CWD** — nested project `.firebaserc` at `apps/myapps/` isn't found by workflows that run from repo root. Root-level alias fixes it cleanly without clobbering nested surfaces.
- **Background subagents can't invoke `/end-subagent-session`** — that's a slash-command skill, not a tool. They signal intent but the parent session's closing-sweep handles the commit. This is already working correctly; noted so next Evelynn doesn't mistake "session close triggered" output for an error.
