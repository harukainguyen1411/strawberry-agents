## Evelynn — 2026-04-20T04:34:50Z

Yes — Claude Code feature, shipped in v2.1.80 as a research preview. Short version:

**Channels** are a way for external local processes to push events *into* a running Claude session. Think of them as the inverse of MCP: MCP lets Claude pull data out (tools Claude calls); Channels lets data flow in (events Claude reacts to) without Claude having to ask.

A channel is a small plugin (Bun/TypeScript) that runs alongside your `claude` process. It emits events, and the running session picks them up when idle — like a notification that Claude can read and decide to act on. Use cases Anthropic highlights: webhooks, file-change notifications, ambient signals.

For our case: the plugin watches `agents/sona/inbox/`, and when you send Sona a message, the plugin emits *"new inbox, run /check-inbox"* — her running session sees it on its next idle moment and responds. That's the "ping" we need.

Official docs: `code.claude.com/docs/en/channels`.
