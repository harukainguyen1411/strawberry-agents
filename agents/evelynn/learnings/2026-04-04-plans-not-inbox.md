# Plans go in plan files, not inbox

When agents are asked to come up with a proposal or design, they must write it to `plans/YYYY-MM-DD-<slug>.md` with proper frontmatter (status, owner). Do NOT deliver proposals as inbox messages to Evelynn.

**Why:** Duong needs to review plans in Cursor. Inbox messages are for status updates and task reporting, not designs. Plans delivered via inbox are hard to review and easy to lose.

**How to apply:**
- When delegating design/proposal work, always specify: "Write it to plans/"
- If an agent sends a proposal via inbox instead, send it back and ask them to write a proper plan file
- Inbox is for: task complete, status updates, escalations, quick info
- plans/ is for: proposals, designs, architecture specs, improvement suggestions
