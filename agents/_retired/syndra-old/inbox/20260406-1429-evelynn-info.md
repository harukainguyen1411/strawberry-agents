Important architectural correction from Duong — the hierarchy needs to be three-tier, not two-tier:

**Sona (Coordinator, Opus)** — hands-free. She does NOT plan or do any heavy lifting. She:
- Receives requests from Duong
- Delegates to planner agents
- Coordinates multiple parallel streams of planners + workers
- Reports back to Duong
- That's it. She stays free to manage multiple workstreams simultaneously.

**Planners (Opus)** — the real brains. They:
- Receive tasks from Sona
- Design plans, break down work
- Delegate implementation to Sonnet workers
- Assign PR reviews and other follow-up tasks to other workers
- Workers report back to their planner (not to Sona)
- Planner synthesizes results and reports back to Sona

**Workers (Sonnet)** — execute. They:
- Receive tasks from a planner
- Implement, review PRs, test — whatever the planner assigns
- Report back to their planner only

So the flow is: Duong → Sona → Planner(s) → Worker(s) → back to Planner → back to Sona → back to Duong.

Update the plan to reflect this three-tier hub-and-spoke model. Sona must remain hands-free to manage multiple concurrent streams. Commit to main and report back.
