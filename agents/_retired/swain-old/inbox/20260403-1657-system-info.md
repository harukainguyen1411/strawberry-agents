Duong wants to build an end-to-end contributor pipeline for his personal projects (strawberry monorepo):

1. Contributors (friends) submit suggestions/ideas/bugfixes via Discord
2. Submission triggers a Claude Code session in the cloud
3. Claude Code picks up the request, makes changes on a branch
4. A preview tunnel is created so the contributor can view their changes live
5. Once the contributor approves, Duong runs the deployment

Your job: Design the technical architecture for this. Consider:
- How to trigger Claude Code sessions from Discord (webhooks, bot, MCP?)
- Where Claude Code runs (cloud VM, GitHub Actions, remote agent?)
- How to create preview tunnels (Cloudflare Tunnel, ngrok, Vercel preview deployments?)
- The approval → deploy flow
- Cost, complexity, and security implications
- What already exists in strawberry (agent-manager MCP, Discord MCP already connected)

Don't implement — propose the architecture. Report back to Evelynn with your recommendation. Keep it concise and opinionated — pick the best path, don't list 10 options.
