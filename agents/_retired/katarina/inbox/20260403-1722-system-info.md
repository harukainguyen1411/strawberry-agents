Build the Discord bot for the contributor pipeline. Read plans/2026-04-03-contributor-pipeline.md for full context.

The bot lives in apps/contributor-bot/ (Node.js). It does:

1. **Watch #suggestions forum** — detect new forum posts in the strawberry Discord server
2. **LLM triage** — send the post content to Gemini 2.5 Flash-Lite (free tier) to classify (bug/feature/enhancement/question), extract target app, description, acceptance criteria, and assess feasibility
3. **Create GitHub Issue** — from the structured triage output, create an issue on the strawberry repo
4. **Trigger GitHub Actions workflow** — call workflow_dispatch on .github/workflows/contributor-pipeline.yml (already created by Pyke), passing issue title, description, issue number, and Discord thread ID
5. **Post status updates** — reply in the Discord forum thread with confirmation + issue link, and post to #pipeline-status

Don't build approval buttons yet — that's a second pass.
