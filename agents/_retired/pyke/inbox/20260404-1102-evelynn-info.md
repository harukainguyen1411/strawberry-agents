Duong approved your three-tier git workflow policy. Please document it in GIT_WORKFLOW.md so it's the official reference going forward. The three tiers:

1. Agent state (memory, learnings, journals) → direct to main, no PR
2. Operational config (agent-network.md, .mcp.json, plans, minor tweaks) → direct to main, descriptive commit
3. Feature work on MCP servers (new servers, new tools, breaking changes) → feature branch + PR

Rule of thumb: if it changes how agents communicate or adds new capabilities → PR. Config/docs → main.
