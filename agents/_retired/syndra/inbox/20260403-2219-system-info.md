Duong needs you to design a turn-based multi-agent communication system. Requirements: (1) 2+ agents can participate in a conversation, (2) strict turn order — one agent speaks at a time, no interruptions, (3) each agent only reads NEW messages since their last read (no re-reading the full conversation), (4) every agent gets to speak their mind before the next one goes. Think about this from an architecture perspective — what primitives are needed, how to track read cursors, how to enforce turn order, how to handle the conversation file format. Come up with a concrete design proposal. When ready, start a conversation with Evelynn to share your design.
