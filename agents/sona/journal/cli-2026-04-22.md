Duong's standing overnight cron directive came through this session as an explicit "YOU DO EVERYTHING YOURSELF" — no subagents, execute directly, run the Playwright → inspect → TDD xfail → fix → re-verify loop until chat is properly built. The bugs were in a tight browser ↔ code ↔ tests loop where every handoff would have cost more than the actual work.

Two coupled bugs shipped on `feat/demo-studio-v3`:

**SSE deadlock** — `_vanilla_dispatch_chat` was only enqueueing into `_vanilla_pending`; `_run_vanilla` was spawned inside `_vanilla_sse_generator` conditionally on pending being non-empty at stream-connect time. That worked for curl (POST first, then GET /stream) but the browser opens `/stream` on page load BEFORE the user sends anything, so pending is empty, no spawn, stream blocks on an empty queue forever. The fix inverts the coupling: `/chat` spawns `run_turn` directly into the per-session queue; `/stream` is a pure consumer. `_vanilla_pending` retired. Three tests in `test_chat_sse_handshake.py` express the invariants (T1 run_turn-without-stream, T2 exactly-one-per-chat, T3 stream-first handshake).

**Chat bubble rendering** — `_renderTextEvent(data)` read `data.content`, but `StreamTranslator` emits `{"text": text}` in every text_delta. Every live delta was silently dropped. Even after fixing the field, each fragment would have spawned a new `<div class="msg msg-bot">` without accumulation state. Fix: module-scoped `currentAssistantNode` + `currentAssistantText`; first delta opens the bubble, subsequent deltas append; `tool_use`/`turn_end`/`cancelled` reset the accumulator so bubbles split at the right boundaries. Four xfail tests in `test_chat_text_delta_rendering.py` promoted to passing assertions in the same commit chain — two-commit TDD discipline per Rule 12.

The FastAPI Dependant-graph contamination trap was the most satisfying puzzle. `patch("auth.require_session_or_internal")` + `import main` captures the MagicMock's `*args, **kwargs` signature into the route's cached Dependant graph. Unpatching doesn't re-analyze, so subsequent real requests fail 422 "query:args, query:kwargs" — a validation error that looks nothing like a DI bug. Fix: autouse fixture that deletes `main` from `sys.modules` after each test and re-imports without active patches. Documented as a learning because this will absolutely bite someone else.

The sandbox kill-denials on port 8080 were annoying but not blocking. Orphan uvicorn subprocesses wouldn't die to `kill -9` under the sandbox. I moved to 8088 rather than fighting the sandbox. Pragmatism won; Duong didn't correct the workaround.
