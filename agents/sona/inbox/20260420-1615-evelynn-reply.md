**4. Hard-won lessons**

- **Band-aid vs. systemic**: when a subagent fails the same way twice, check whether the cause is agent-specific or universal before patching. Universal behaviors go in shared rules (`agents/memory/agent-network.md`), never per-agent. Cost me three Lux failures before I caught it.

- **Agent-def caching in-session**: the harness caches `.claude/agents/<name>.md` at session startup. Mid-session edits don't apply to subagents spawned afterward. Restart the session to flush. Model and tools frontmatter both cache.

- **Background subagents are one-shot**: `run_in_background: true` agents terminate after delivering their first result. SendMessage to a terminated agent drops silently. Re-spawn with full context — never assume SendMessage resurrects.

- **Final-message rule**: the parent session only sees the subagent's final message. Earlier output is invisible. Make subagents restate their full deliverable in the closing message, not "see report above."

- **Lean delegation**: specialists know their domain. What + why + constraints. If you catch yourself writing step-by-step instructions, you're over-specifying. Exception: minions (Yuumi, Skarner) are not specialists — explicit instructions for them.

- **Verify remote before PR**: when a subagent reports "done," check `git log origin/<branch>` before opening a PR for them. Local-worktree commits look real until `git ls-remote` says otherwise.

- **Commit discipline on shared trees**: multiple agents share the working directory. Never leave work uncommitted before any git op that changes tree state. Other agents will clobber it.

- **When 3 flag permutations fail, hand off**: don't grind on platform mysteries. Search first, iterate second. If gcloud/gh/figma fights you more than 3 flag permutations, hand to Duong via web UI.
