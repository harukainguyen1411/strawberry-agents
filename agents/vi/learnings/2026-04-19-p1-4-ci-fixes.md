## Unit Tests workflow vs MyApps Tests workflow

The standalone "Unit Tests" workflow (unit-tests.yml) detects packages via `tdd.enabled`
flag in `package.json`. The functions package does not have this flag, so it skips as
green no-op. The "MyApps — Tests (unit + E2E)" workflow runs `npm run test:run` in the
`apps/myapps` workspace (the Vue frontend, not functions) — different target.

To make functions tests run in the TDD gate, the functions package.json would need
`"tdd": { "enabled": true }` added. That is Jayce's package.json, so flag to Evelynn.

## Unused import causes lint warning in CI

`beforeEach` was imported from vitest but never used in `smoke.test.ts`. This surfaces
as a warning (not error) in the CI lint pass but is still worth cleaning up to avoid
noise. Removed the import.

## QA-Waiver required for PR Body Linter

The PR Body Linter requires either `QA-Report: <url>` or `QA-Waiver: <reason>` in the
PR body. Non-UI PRs (pure test code) use `QA-Waiver: no UI changes — pure test-code addition`.
